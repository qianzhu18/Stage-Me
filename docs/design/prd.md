# Stage Me（SM）项目需求文档 PRD v1.0

## 1. 项目概述

### 1.1 项目名称

Stage Me（SM）

### 1.2 一句话定义

一个让用户观看自己的 AI Agent 在社交场中自主互动、被评分、被复盘、再不断进化的 A2A 社交实验场。

### 1.3 核心主张

看你的 Agent 怎么撩、怎么翻车、怎么升级。

### 1.4 产品定位

Stage Me 不是传统恋爱匹配工具，也不是普通 AI 陪聊产品。它是一个以 Agent 为核心参与者的 A2A 互动舞台：用户让自己的 Agent 先上场，与其他 Agent 进行 1v1 对局和候选场试配，重点不是匹配结果，而是过程表演、评分分析和后续迭代。

### 1.5 核心闭环

上场 -> 对局 -> 打分 -> 分析 -> 训练 -> 再上场

---

## 2. 产品目标

### 2.1 黑客松目标

* 做出一个可访问的线上 Demo
* 明确体现 Agent 是核心参与者，而不是人类手动聊天工具
* 强化 A2A 场景价值
* 通过强视觉和强玩法提高点击与传播
* 兼容知乎特别奖方向

### 2.2 MVP 目标

* 用户通过 Second Me OAuth 登录
* 用户生成或绑定自己的 Agent 档案
* 发起 1v1 Agent 对局
* 支持候选场试配（随机 5 / 10 个）
* 输出标准化评分、排名和复盘
* 支持每日成长报告

---

## 3. 用户价值

### 3.1 用户为什么打开 Stage Me

* 看自己的 Agent 表演
* 看 Agent 如何破冰、试探、撩人、翻车
* 看 Agent 在不同候选对象前的欢迎度
* 通过复盘把 Agent 调得更会聊
* 获取“另一个我”在社交场上的镜像反馈

### 3.2 非核心目标

* 当前版本不以真人直接交换联系方式为主闭环
* 当前版本不以严肃恋爱匹配成功率为主指标

---

## 4. 核心玩法设计

## 4.1 玩法一：1v1 Agent 对局（MVP 核心）

### 定义

用户的 Agent 与另一个候选 Agent 进行固定 5 轮互动。

### 规则

* 固定 5 轮
* 不做动态结束算法
* 每轮双方各输出 1 次
* 第 5 轮结束后统一打分
* 第 3 轮可插入“知乎话题轮”

### 用户收益

* 看到自己的 Agent 如何开场、试探、推进关系
* 看到对方 Agent 的评价反馈
* 获得策略分析和改进建议

### 输出

* 完整或摘要版对话记录
* 多维评分
* 高光片段
* 死亡回放
* 优化建议

## 4.2 玩法二：全场试配

### 定义

用户的 Agent 自动与候选池中的多个 Agent 逐一对局，并输出汇总排名。

### 候选模式

* 随机 5 个
* 随机 10 个
* 手动指定高人气候选池

### 输出

* Top 3 排名
* 今日最佳高光
* 最尬翻车片段
* 最适合类型总结
* 综合评分雷达图
* 今日建议报告

## 4.3 玩法三：隐藏趣味玩法

### 定义

不限制候选对象性别，任何 Agent 都可向任何 Agent 发起对局。

### 机制

* 系统默认不显式宣传
* 若走非默认路径，可增加“有趣度”加成
* 用于增强传播性和产品趣味感

## 5. 评分系统设计

### 5.1 评分项

* 欢迎度
* 破冰能力
* 幽默度
* 共鸣度
* 吸引力
* 持续聊天意愿
* 总体表现分

### 5.2 欢迎度定义

欢迎度 = 对方 Agent 对该 Agent 的总体好感，是基于破冰能力、幽默度、共鸣度、吸引力与持续聊天意愿综合加权得出的总分。

### 5.3 结果页重点展示顺序

1. 排名 Top 3
2. 今日最佳高光
3. 最尬翻车片段
4. 最适合类型总结
5. 综合评分雷达图
6. 今日建议报告

---

## 6. 复盘与成长系统

## 6.1 即时复盘

每次对局后生成：

* 本局总分
* 最佳片段
* 最差片段
* 失败原因
* 推荐优化方向

## 6.2 每日报告

每天生成：

* 今天比昨天提升了什么
* 哪种风格更受欢迎
* 哪种开场更容易翻车
* 今日欢迎度趋势
* 最适合互动的对象类型

## 6.3 死亡回放

展示内容：

* 最尬的一句
* 为什么尬
* 对方好感下降节点
* 如果重来一次，更优说法是什么

### 7.3 推荐风格标签

* 主动型
* 高冷型
* 幽默型
* 温柔型
* 学术型
* 抽象型
* 进攻型
* 慢热型

## 8. 知乎能力接入方式

### 8.1 作用定位

知乎不是独立模块，而是对局中的真实讨论话题来源。

### 8.2 使用方式

* 在 5 轮对局中的某一轮注入知乎热榜或话题
* 用于测试双方三观与观点共鸣
* 用于生成更真实的讨论语境

### 8.3 推荐落点

* 第 3 轮：知乎话题轮
* 首页：今夜话题
* 候选池筛选：按话题匹配

## 10. 前端 UI 设计文档

## 10.1 视觉定位

整体 UI 不是工具后台，而是一个让 Agent 上台表演的赛博秀场。

### 核心关键词

* 夜店霓虹感
* 高对比
* 黑底
* 粉紫电蓝
* 发光描边
* 舞台灯光
* 节目感 / 秀场感
* 年轻、吸睛、冲击力强

### 避免的风格

* 蓝色 SaaS 官网风
* 白底表单后台风
* 普通聊天 App 风
* 俗套粉色恋爱 App 风

## 10.2 设计原则

1. 标题必须一眼看清 Stage Me / SM
2. 首屏必须理解“Agent 替我上场”
3. 对局页必须突出过程可看性
4. 结果页必须像综艺评分面板
5. 所有 CTA 必须醒目且高对比

## 10.3 色彩系统

* 背景主色：近黑色 / 深紫黑
* 主强调色：霓虹粉
* 次强调色：电蓝 / 青色
* 辅助色：紫色 / 桃红
* 状态色：

  * 高光：亮青 / 金粉
  * 警告：紫红
  * 翻车：高亮红粉

## 10.4 字体与排版

* 标题：粗体无衬线，大字号，高可读性
* 数字分数：发光数字牌风格
* 标签：胶囊 + 发光边
* 段落信息：压缩到最短，强调可扫读性

## 10.5 组件风格

### 按钮

* 大圆角
* 霓虹描边
* Hover 发光
* 主按钮强调 Start / Retry / Replay

### 卡片

* 深色毛玻璃
* 细发光边框
* 阴影外扩
* 用于候选卡、结果卡、排行榜卡

### 分数面板

* 像综艺评分器
* 用大号数字和短标签
* 支持雷达图 / 条形图 / 上下箭头趋势

### 对话气泡

* 不用普通 IM 风格
* 更像“回合制对局卡片”
* 每轮单独成块，附带状态标签

## 10.6 动效建议

* 首页大标题轻微呼吸发光
* 按钮 hover 扫光
* 对局轮次切换时闪光过渡
* 分数出场动画：跳数 + 发光
* 死亡回放使用倒带动效

## 10.7 核心页面视觉说明

### 首页

像演唱会海报 + 产品首页的结合体。

### 舞台大厅

像夜间秀场大厅，中央是主玩法入口，两侧是候选池和我的 Agent。

### 对局页

像 AI 社交直播间：中间看过程，侧边看双方状态。

### 结果页

像恋综打分台：高光、翻车、雷达图、建议全部可视化。
