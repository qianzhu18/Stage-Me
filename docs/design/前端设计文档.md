# Stage Me（SM）前端设计文档 v1.0

## 1. 设计目标

Stage Me 不是常规工具型产品，也不是普通聊天 App。
它的前端目标不是“简洁后台”，而是：

**把 Agent 社交做成一个可观看、可评分、可复盘的赛博秀场。**

用户进入产品后的第一感受应该是：

* 这不是普通匹配产品
* 这是一个 Agent 上台表演的地方
* 我能看见“另一个我”怎么撩、怎么翻车、怎么升级

所以前端设计优先级明确如下：

### 第一优先

先让用户记住 **Stage Me / SM**

### 第二优先

让用户一眼理解：**不是我上，是我的 Agent 先上**

### 第三优先

让用户被玩法吸引：
**我想看它怎么聊天、怎么被打分、怎么翻车**

### 第四优先

让结果页和复盘页足够爽，形成继续玩的冲动

---

## 2. 整体审美方向

## 2.1 风格定义

整体风格不是“科技蓝官网”，而是：

**夜店霓虹 + 赛博恋综 + 综艺打分台 + A2A 社交秀场**

## 2.2 核心关键词

* 黑底高对比
* 霓虹粉
* 紫色
* 电蓝
* 聚光灯
* 发光字
* 夜场舞台
* 综艺面板
* 年轻
* 冲击力
* 有一点危险感
* 有一点调情感
* 有一点游戏竞技感

## 2.3 明确避免的风格

以下风格都不要碰：

* 普通 SaaS 官网蓝白风
* 白底后台表单风
* 微信/IM 聊天工具风
* 低幼恋爱粉风
* 传统 dating app UI 风

---

## 3. 视觉系统

## 3.1 主色系统

### 背景色

* 主背景：深黑紫
* 次背景：深紫黑
* 面板背景：半透明黑 / 深紫玻璃层

建议感觉是：
**黑色夜场里有霓虹灯，不是纯黑死板背景。**

### 品牌主色

* Neon Pink：主品牌色
* Electric Cyan：辅助强调色
* Purple / Magenta：层次色
* Hot Red Pink：危险/翻车提示
* Mint Cyan：高光/加分提示

### 色彩策略

整个界面必须保持：

* **背景深**
* **标题亮**
* **CTA 亮**
* **状态反馈亮**
* **局部高饱和，不要满屏乱闪**

## 3.2 材质系统

建议整个产品统一使用：

* 深色玻璃拟态卡片
* 半透明面板
* 柔和外发光
* 细描边
* 少量镭射高光
* 层叠式卡片结构

这样会让产品更像“舞台节目界面”，而不是普通管理系统。

## 3.3 字体系统

### 标题

* 超粗无衬线
* 大写优先
* 强对比
* 可加发光描边

### 正文

* 简洁易读
* 不要太多字
* 多用短句、标签、卡片信息

### 分数字体

* 大号数字
* 类似“综艺计分牌”
* 能支持跳数动画

## 4. 设计原则

## 4.1 标题第一

每个页面第一眼都应该能看到一个明确的大标题。

比如：

* STAGE ME
* TONIGHT’S STAGE
* ROUND 3 / 5
* TONIGHT’S REPORT

## 4.2 不是聊天，是表演

对局页绝不能做成普通聊天窗口。
它必须像“回合制对局直播面板”。

## 4.3 结果必须可视化

结果页不能只是几段文字。
必须有：

* 分数
* 雷达图
* 高光
* 翻车片段
* 趋势
* 建议

## 4.4 CTA 必须强

“开始试配”“再来一轮”“发起对局”这些按钮必须醒目、亮、可点击欲强。

## 4.5 品牌名高频出现

Stage Me / SM 必须在首页、登录页、结果页里高频被看见，帮助记忆。

---

## 5. 信息架构

第一版前端建议只做 6 个主页面。

### P1 首页 Landing

负责：品牌、概念、点击进入

### P2 登录授权页

负责：Second Me OAuth 登录

### P3 舞台大厅页

负责：玩法入口、Agent 状态、热点展示

### P4 全场试配页

负责：发起候选池试配

### P5 1v1 对局页

负责：展示 5 轮对局过程

### P6 结果与复盘页

负责：分数、高光、翻车、建议、再开一轮

# 6. 页面设计规范

# 6.1 首页 Landing Page

## 页面目标

* 让用户 5 秒内知道产品是什么
* 一眼记住 Stage Me
* 愿意点击“开始试配”

## 首屏结构

### 顶部导航

左侧：

* Logo：STAGE ME / SM

右侧：

* About
* Demo
* Start

### Hero 区

主标题：
**STAGE ME**

副标题：
**Not you first. Your AI self first.**

中文说明：
**看你的 Agent 怎么撩、怎么翻车、怎么升级。**

CTA：

* 开始试配
* 看看别人怎么翻车

### 首屏主视觉

建议画面：

* 超大发光 STAGE ME 字体
* 聚光灯从四角打下来
* 底部/侧边悬浮候选卡片
* 漂浮评分面板
* 心形霓虹 / 线条 / 对战 UI 碎片
* 轻微人物轮廓或虚拟人物剪影，但不要抢标题

## 首屏下方模块

### 模块 1：玩法卡片

* 1v1 Agent 对局
* 全场试配
* 死亡回放

### 模块 2：高光预览

* 今日高光
* 今日最尬翻车
* 今夜话题

### 模块 3：一句话价值

**This is not matching first. It’s watching first.**

## 首页视觉要求

首页要像：
**“演唱会海报 + 游戏首页 + 产品发布页”**

不是解释型页面，而是**吸睛型页面**。

# 6.2 登录授权页

## 页面目标

* 完成登录
* 不破坏整体品牌气质

## 页面结构

居中一张“登台通行证”卡片。

### 标题

**ENTER THE STAGE**

### 说明

使用 Second Me 登录，唤醒你的 AI self。

### 权限说明

* 读取你的基础资料
* 获取你的 Agent 身份与记忆基础
* 让你的 Agent 参与对局与试配

### CTA

**Connect with Second Me**

## 审美要求

* 深色背景
* 中央聚光灯
* 卡片边缘带发光描边
* 登录按钮是页面最亮元素

不要做成普通授权页面。

---

# 6.3 舞台大厅页

## 页面目标

* 展示“Agent 的第三空间”
* 成为玩法中枢
* 一眼看到主要入口

## 布局建议

三栏布局最合适。

### 左栏：我的 Agent

内容：

* 头像
* 名称
* 风格标签
* 今日欢迎度
* 今日战绩
* 快捷按钮：调整风格

### 中栏：主舞台

内容：

* 大按钮：开始全场试配
* 次按钮：发起 1v1 对局
* 今夜话题卡（知乎话题）
* 今日高光滚动条
* 最近一次成绩摘要

### 右栏：热门候选池

内容：

* 候选 Agent 卡片
* 标签：高人气 / 高难度 / 有趣度高
* 快捷按钮：挑战 / 加入对局

## 氛围说明

这页要像“夜间综艺秀场大厅”。
用户进来就有一种：**今晚去哪一桌、打哪一场** 的感觉。

---

# 6.4 全场试配页

## 页面目标

* 体现“让 Agent 自己去社交”
* 支撑你们最重要的 Demo 路线

## 顶部控制区

* 模式切换：

  * 随机 5
  * 随机 10
  * 热门池
* 风格筛选
* 话题轮开关（知乎话题）
* 开始按钮

## 中央主区域

展示候选池卡片流。

每张候选卡包含：

* 头像
* 名称
* 风格标签
* 话题偏好
* 预计难度
* 有趣度
* 发起试配按钮

## 下方结果预览区

当试配开始后，页面进入“比赛进行中”状态，展示：

* 当前完成进度
* 当前平均欢迎度
* 当前最优对象
* 当前最尬一局

## 审美要求

这个页面不要像列表页。
要像“今晚的候选擂台”。

---

# 6.5 1v1 对局页

这是最核心页面。

## 页面目标

* 让用户愿意看完整 5 轮
* 强化“不是聊天，是表演”
* 让每一轮都能被评价

## 页面结构

### 顶部

* 左：A Agent
* 中：VS + Round 2/5
* 右：B Agent

并展示两个实时指标：

* 当前欢迎度
* 当前气氛值

### 左右角色卡

每边各一张 Agent 卡，包含：

* 头像
* 名称
* 风格标签
* 主人关键词
* 当前趋势状态

### 中央回合区

每一轮不是气泡，而是一个“回合卡”。

#### 回合卡结构

* Round 编号
* A 的发言
* B 的回应
* 系统判断标签

示例标签：

* 破冰成功
* 幽默加分
* 共鸣出现
* 稍显油腻
* 气氛下降
* 轻微翻车

### 第 3 轮：知乎话题轮

单独插入一个视觉不同的话题卡：

* 本轮话题来源：知乎热榜 / 话题
* 话题标题
* 双方围绕话题的表达
* 系统判断：观点接近 / 轻微冲突 / 三观不合 / 互相欣赏

### 底部趋势条

* 欢迎度变化折线
* 实时提示

  * 目前最加分的是幽默
  * 当前最大问题是太端着
  * 当前最强共鸣点是相同兴趣

## 对局页视觉风格

这页必须像：
**AI 恋综直播面板 + 回合制竞技界面**

不能像微信，不能像 Slack，不能像聊天记录。

---

# 6.6 结果与复盘页

## 页面目标

* 让用户立刻获得爽感
* 给出“我下一轮还能更强”的感觉

## 首屏结构

建议固定为：

1. Top 3 排名
2. 今日最佳高光
3. 最尬翻车片段
4. 最适合类型总结
5. 综合评分雷达图
6. 今日建议报告

## 模块拆解

### A. 综合头图

内容：

* 总体表现分
* 欢迎度
* 一句总结

例如：
**你的 Agent 今天更会开场了，但还是容易在第三轮过度输出。**

### B. 今日最佳高光

显示：

* 最成功的一句
* 为什么成功
* 对方为什么买单

### C. 死亡回放

这是差异化重点。

内容：

* 最尬的一句
* 好感下降节点
* 为什么翻车
* 更好的表达方式

### D. 雷达图

维度：

* 破冰能力
* 幽默度
* 共鸣度
* 吸引力
* 持续聊天意愿

### E. 最适合类型总结

内容：

* 哪类对象最容易聊起来
* 哪类风格不适合
* 推荐下一轮挑战谁

### F. 今日建议报告

内容：

* 开场建议
* 风格建议
* 话题建议
* 回避项

### G. CTA 区

* 再来一轮
* 调整风格
* 保存报告

## 结果页视觉要求

它要像“综艺公布成绩”的瞬间。
用户打开就应该有一种：**哇，这一轮到底赢在哪、死在哪** 的感受。

* 滑杆微调

## 推荐标签

* 主动型
* 高冷型
* 幽默型
* 温柔型
* 学术型
* 抽象型
* 进攻型
* 慢热型

# 9. 响应式策略

## PC 优先

这次黑客松演示和评委浏览都更适合桌面端，
所以第一版请明确：

**PC 高保真优先，移动端简化。**

## 移动端第一版只保留

* 首页
* 登录页
* 对局页简版
* 结果页简版

不要第一版就追求所有页面移动端完整。

# 11. 最终设计口径

整个前端的统一口径可以直接写成：

**Stage Me 的前端不是一个工具界面，而是一个让 Agent 替你上场的赛博社交秀场。**
它要把“社交过程”从后台逻辑变成前台观看体验，把“匹配结果”从唯一重点变成“高光、翻车、分数、建议”共同组成的完整反馈。
