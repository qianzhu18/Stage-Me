# 📋 文档管理心法速查卡

> 一语道破：**指令驱动，做完即走，知识沉淀，时间封存**

---

## 🗂️ 三大文件夹

```
stage me/
├── commands/          # ⏳ 待办池 - 有想法就扔进去，做完就清空
│   ├── design/        # 设计类任务
│   ├── fixes/         # 修复类任务
│   └── other/         # 其他任务
├── design/            # 💎 设计知识库 - 沉淀下来的真理
├── fixes/             # 🔧 问题记录库 - 踩过的坑
├── sql/               # 🗄️ SQL脚本库
└── history/           # 📦 时间线 - 按日期封存历史
    └── YYYYMMDD/
```

---

## 🔄 工作流

```
想法 → commands/ → AI执行 → 分类归档
                      ↓
        ┌────────────┼────────────┐
        ▼            ▼            ▼
     fixes/      design/      history/
     问题         知识         历史
```

---

## 📝 命名公式

| 类型 | 公式 | 示例 |
|------|------|------|
| 指令 | `动作-对象.md` | `编码-添加登录功能.md` |
| 问题 | `问题描述.md` | `activity_tracking_analysis.md` |
| 归档 | `YYYYMMDD/` | `20250316/` |
| 脚本 | `序号_功能.sql` | `001_tables.sql` |

---

## 🤝 人机分工

| 你做什么 | AI做什么 |
|----------|----------|
| ✍️ 创建指令文件 | 📖 读取并执行指令 |
| 👀 审核结果 | 💻 生成代码/文档 |
| 📂 决定归档时机 | 🗂️ 整理过程记录 |

---

## ⚡ 最少操作

1. **写指令** → `commands/`
2. **给路径** → 让AI执行
3. **看结果** → 调整/确认
4. **移历史** → `history/日期/`

---

## 🎯 实践检查清单

每次任务完成后检查：
- [ ] commands/ 中的已完成任务是否已移出？
- [ ] 产生的问题是否记入 fixes/？
- [ ] 有价值的知识是否存入 design/？
- [ ] 完成的历史是否归档到 history/YYYYMMDD/？
